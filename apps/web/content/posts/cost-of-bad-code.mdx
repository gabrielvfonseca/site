title: "The Real Cost of Bad Code"
description: "Bad code isn't cheap. You just pay for it later, with interest."

---

Bad code feels fast. That's the trap. You skip the edge case, you copy-paste instead of abstracting, you ship. The deadline is met and everyone's happy. The bill just hasn't arrived yet.

It always arrives. And it's never smaller than what you saved.

## You don't save time, you borrow it

The shortcut that saved you an hour on Tuesday costs three hours in December, when someone — often you — has to understand it again, work out what it was supposed to do, and change it without breaking the five things quietly depending on it. Speed you can't sustain isn't speed. It's debt with a variable interest rate.

The worst part is that the interest compounds silently. Nobody sends you an invoice. The codebase just gets a little slower to change, a little scarier to touch, until one day a one-line feature takes a week and no one can quite say why.

## Bad code is a bug factory

Most bugs aren't clever. They're the natural output of code that's hard to read. When you can't hold a function in your head, you can't see what it does wrong. Unclear code doesn't just contain bugs — it manufactures them, because every change is a guess.

```javascript
// The classic
function average(nums) {
  let sum = 0;
  for (let i = 0; i < nums.length; i++) sum += nums[i];
  return sum / nums.length; // divides by zero on an empty array
}
```

The fix isn't cleverness. It's caring enough to handle the empty case:

```javascript
function average(nums) {
  if (nums.length === 0) return 0;
  return nums.reduce((a, n) => a + n, 0) / nums.length;
}
```

Same amount of code. One of them just decided to be honest about reality.

## Clarity is the actual skill

Anyone can make code that works once. The job is making code the next person can safely change — and the next person is usually you, six months from now, with no memory of what you were thinking. Clear names, small functions, and handling the case you'd rather ignore aren't polish. They're the work.

I'd rather write the boring, obvious version now than pay for the clever one later. The boring version is cheaper. It's just cheaper in a currency you don't feel until the bill comes.
