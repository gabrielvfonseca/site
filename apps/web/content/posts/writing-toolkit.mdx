Plain prose carries most of the weight on this site, but some ideas want more than a paragraph. Over time I've built a small kit of MDX components that stay out of the way and match the rest of the design. This post is both a reference and a live demo of that kit.

## Callouts

Callouts pull a single point out of the flow. There are five variants, monochrome by default with a destructive accent reserved for the ones that need it.

<Callout variant="note">
  A neutral aside — context that's useful but not urgent.
</Callout>

<Callout variant="tip" title="Worth knowing">
  Tips get a title override and a lightbulb. Good for shortcuts.
</Callout>

<Callout variant="success" title="Shipped">
  Confirmations and wins land here.
</Callout>

<Callout variant="warning">
  Warnings borrow the destructive token — use them sparingly.
</Callout>

<Callout variant="danger" title="Don't do this">
  The strongest signal, for genuinely dangerous footguns.
</Callout>

## Steps

For walkthroughs, `Steps` renders an auto-numbered process with a connector down the gutter.

<Steps>
  <Step title="Draft in Markdown">
    Write the words first. Structure and components come after the argument works.
  </Step>
  <Step title="Reach for a component">
    Only when prose stops being enough — a metric, a comparison, a callout.
  </Step>
  <Step title="Read it back on mobile">
    If it doesn't hold up in a narrow column, it doesn't ship.
  </Step>
</Steps>

## Tabs

When one explanation has two audiences, `Tabbed` keeps them side by side without doubling the page length.

<Tabbed
  items={[
    { label: "For engineers", content: "The API is a single typed function. No config, no runtime surprises, exhaustive tests." },
    { label: "For everyone else", content: "It just works, and it keeps working when the load shows up." },
  ]}
/>

## Code, with a copy button

Every fenced code block gets a header and a one-tap copy affordance.

```ts title="reading-time.ts"
export function getReadingTime(words: number): number {
  return Math.max(1, Math.round(words / 220));
}
```
