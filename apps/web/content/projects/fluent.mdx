Fluent is an editor theme designed around a single design goal: make the code the loudest thing on the screen. That sounds obvious, but most themes violate it by drawing attention to UI chrome, syntax highlights that scream at different volumes, or decorative elements that never quite settle into the background.

Fluent is quiet on purpose. Colors are desaturated just enough to recede. Contrast ratios are balanced so that semantic distinction works without visual competition. The palette is narrow with fewer colors than most themes, each one chosen for legibility and information, not aesthetics alone. The result is an environment where you stop noticing the theme entirely and only see the code.

## Design principles

- **Low intra-line contrast**: Token colors differ by hue more than brightness, so highlighted syntax reads as categorization, not emphasis.
- **Deliberate chroma**: Every color in the palette earns its place. If a semantic category can share a color with another, it does.
- **No visual noise**: No unnecessary borders, no glowing selections, no drop shadows. UI elements are distinguished by spacing and weight, not decoration.

## Platform support

Fluent ships for VS Code, Antigravity, and Cursor. Each port respects the platform's native UI conventions while maintaining the core palette and contrast ratios. The experience should feel native to each editor, not like a theme forcibly transplanted.

## Why a theme matters

You look at your editor more than almost anything else you use. A theme isn't decoration but the environment you think in for hours every day. Getting that environment right, getting it to disappear, is a craft problem worth solving carefully.
